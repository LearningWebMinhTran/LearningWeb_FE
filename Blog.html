Chains & Routing

Mục tiêu học tập

Kết nối các bước xử lý tuần tự hoặc rẽ nhánh để xây dựng logic phức tạp. Hiểu các pattern: Simple Chain, Router Pattern, và Conditional Chains.

Tổng quan

Chains & Routing cho phép bạn kết nối nhiều bước xử lý với LLM để tạo ra workflows phức tạp. Simple Chain xử lý tuần tự, Router Pattern phân loại intent, và Conditional Chains có logic điều kiện.

Nội dung chính

  • Simple Chain: Xử lý tuần tự qua nhiều bước
  • Router Pattern: Phân loại intent và route đến model phù hợp
  • Conditional Chains: Logic điều kiện trong chain
  • Best Practices
inventory_2

Áp dụng cho dự án: Personal Knowledge Base

Xây dựng workflow chain và routing logic để xử lý các loại queries khác nhau

Các bước thực hiện

  • Design chain: retrieve relevant notes → generate answer with citations → suggest related topics → format response
  • Implement intent classification: phân loại query thành search, summarize, create_note, general_qa
  • Routing logic: mỗi intent → handler riêng với prompt và processing logic phù hợp
  • Chain execution: sequential processing với error handling ở mỗi step
  • Add conditional logic: nếu không tìm thấy relevant notes → suggest creating new note
  • Test với 20 queries mỗi intent để đảm bảo routing chính xác và chain hoạt động tốt

Kết quả mong đợi

Hệ thống có thể tự động điều hướng người dùng đến đúng chức năng dựa trên câu hỏi tự nhiên, giảm thiểu lỗi và tăng trải nghiệm người dùng.
